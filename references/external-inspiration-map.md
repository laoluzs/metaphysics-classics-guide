# 外部优秀项目可吸收点整理

这个文件用于记录同类开源项目里值得吸收的优点，方便后续持续迭代。

## 1. yi-basic
来源：`westernwaterfall/yi-basic`

### 值得吸收的点
- 资料型项目定位清晰：强调“术数基础”“书目资源”“传承整理”
- 对古籍、博客、论坛、排盘工具做了资源归档
- 适合补强你当前项目的“学习入口”和“延伸阅读”价值

### 建议吸收方式
- 增加一个“扩展资源”参考文件
- 把公开可用的古籍资源、数字化古籍网站、研究入口整理出来
- 让 skill 不只会分析，还能告诉用户去哪里继续学

## 2. bazi-calculator-by-alvamind
来源：`alvamind/bazi-calculator-by-alvamind`

### 值得吸收的点
- README 结构成熟：功能、安装、快速开始、限制、免责声明很完整
- 把“分析能力”和“技术能力”拆开写，层次清楚
- 有明确的“计算引擎”思路，便于后续程序化扩展

### 建议吸收方式
- 你的 README 可继续参考其“Features / Installation / Usage / Notes”结构
- 后续如果你要升级为更强的排盘 skill，可以单独增加“计算层 / 规则层 / 解读层”三层设计说明
- 可增加“未来路线图”，提前说明后续会支持更强的排盘或规则校验

## 3. AI-Fengshui
来源：`sinshine/AI-Fengshui`

### 值得吸收的点
- 风水方向表达简单直接，不绕
- 清楚承认某些计算实现还不够严谨
- 有“先抓核心，不陷入口诀堆砌”的思路

### 建议吸收方式
- 在风水模板里优先讲形势、动线、空间关系，而不是堆术语
- 在 README 或后续 roadmap 里明确哪些能力是“分析草稿”，哪些能力是“后续计划”
- 对算法未完全实现的部分保持透明

## 建议融入到当前项目的三条线

### A. 资料层
从 yi-basic 吸收：
- 古籍资源入口
- 延伸阅读入口
- 研究型定位

### B. 结构层
从 bazi-calculator-by-alvamind 吸收：
- README 结构成熟度
- 计算层 / 分析层分离思路
- 明确的免责声明和限制说明

### C. 风格层
从 AI-Fengshui 吸收：
- 风水表达更直接
- 不装全能
- 实事求是写清边界
